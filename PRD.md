# PRD: 강사 관리 CRM (Admin)

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 강사 관리 CRM |
| 목적 | 강사 정보를 등록·수정·조회하고 현황을 대시보드로 파악하는 관리자 시스템 |
| 범위 | 프론트엔드 퍼블리싱 (API 연동 없이 UI만 구현) |
| 대상 사용자 | 내부 관리자 |

---

## 2. 페이지 구성

### 2.1 강사 대시보드 (`/`)

대시보드 페이지에서 강사 현황을 한눈에 파악한다.

#### 위젯

| # | 위젯 | 표시 데이터 | UI 형태 |
|---|------|------------|---------|
| 1 | 전체 강사 수 | 총 강사 인원 | 숫자 카드 |
| 2 | 분야별 강사 수 | 30개 활동분야별 인원 | 가로 막대 차트 |
| 3 | 성별 비율 | 남/여 비율 | 도넛 차트 |
| 4 | 등급별 강사 수 | A / B / C / D 등급 인원 | 파이 차트 또는 숫자 카드 |

#### 와이어프레임 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  [전체 강사 수]  [등급 A] [등급 B] [등급 C] [등급 D]      │
├──────────────────────────┬──────────────────────────────┤
│  분야별 강사 수 (Bar)     │  성별 비율 (Donut)            │
│                          │                              │
│                          │                              │
└──────────────────────────┴──────────────────────────────┘
```

---

### 2.2 강사 리스트 (`/instructors`)

전체 강사 목록을 테이블로 표시한다.

#### 기능

| # | 기능 | 설명 |
|---|------|------|
| 1 | 강사 생성 버튼 | 우측 상단 `+ 강사 등록` 버튼 → `/instructors/new` 이동 |
| 2 | 테이블 | 강사 목록 표시 |
| 3 | 강사 ID 클릭 | 해당 강사 수정 페이지 `/instructors/:id/edit` 이동 |
| 4 | 검색 | 이름, 활동분야로 필터링 |
| 5 | 페이지네이션 | 페이지당 20건 |

#### 테이블 컬럼

| 컬럼 | 설명 |
|------|------|
| ID | 강사 고유번호 (클릭 시 수정 페이지 이동) |
| 프로필 | 프로필 사진 썸네일 |
| 이름 | 강사 이름 |
| 전화번호 | 연락처 |
| 활동분야 | 주요 활동분야 (태그) |
| 활동지역 | 주요 활동지역 |
| 등급 | A / B / C / D |
| 등록일 | 등록 날짜 |

---

### 2.3 강사 생성 (`/instructors/new`)

신규 강사를 등록하는 폼 페이지.

### 2.4 강사 수정 (`/instructors/:id/edit`)

기존 강사 정보를 수정하는 폼 페이지. 생성 폼과 동일 구조.

#### 입력 필드

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|------|------|
| 1 | 이름 | text | ✅ | 강사 실명 |
| 2 | 전화번호 | tel | ✅ | 하이픈 자동 포맷 (010-0000-0000) |
| 3 | 이력서 | file (PDF) | ❌ | PDF 업로드 |
| 4 | 포트폴리오 | file (PDF/이미지) | ❌ | 다중 파일 업로드 |
| 5 | 프로필 사진 | file (이미지) | ❌ | 이미지 미리보기 지원 |
| 6 | 활동분야 | multi-select | ✅ | 30개 분야 중 복수 선택 |
| 7 | 활동지역 | multi-select | ✅ | 시/도 단위 복수 선택 |
| 8 | 경력 | textarea | ❌ | 자유 입력 (경력사항) |
| 9 | 자격증 | dynamic list | ❌ | 자격증명 + 발급기관 + 취득일 (동적 추가/삭제) |
| 10 | 학력 | dynamic list | ❌ | 학교명 + 전공 + 졸업연도 (동적 추가/삭제) |
| 11 | 계좌번호 | text | ❌ | 은행명 선택 + 계좌번호 입력 |
| 12 | 신분증 사본 | file (이미지) | ❌ | 이미지 업로드 |
| 13 | 등급 | select | ✅ | A / B / C / D |
| 14 | 성별 | radio | ✅ | 남 / 여 |

#### 폼 레이아웃

```
┌─────────────────────────────────────────────────┐
│  강사 등록 / 수정                                 │
├─────────────────────────────────────────────────┤
│                                                 │
│  [프로필 사진 업로드]                              │
│                                                 │
│  ── 기본 정보 ──────────────────────────────────  │
│  이름:  [__________]    성별: ○남  ○여            │
│  전화번호: [__________]  등급: [A ▾]              │
│                                                 │
│  ── 활동 정보 ──────────────────────────────────  │
│  활동분야: [태그 멀티셀렉트]                       │
│  활동지역: [태그 멀티셀렉트]                       │
│                                                 │
│  ── 경력 & 자격 ─────────────────────────────── │
│  경력:  [textarea]                               │
│  자격증: [+ 추가] [자격증명] [발급기관] [취득일]    │
│  학력:  [+ 추가] [학교명] [전공] [졸업연도]        │
│                                                 │
│  ── 서류 ───────────────────────────────────── │
│  이력서:     [파일 업로드]                         │
│  포트폴리오:  [파일 업로드 (다중)]                  │
│  신분증 사본: [파일 업로드]                        │
│                                                 │
│  ── 정산 정보 ──────────────────────────────── │
│  은행: [선택 ▾]  계좌번호: [__________]           │
│                                                 │
│  [취소]                            [저장]         │
└─────────────────────────────────────────────────┘
```

## 7. 라우팅 구조

```
/                        → 대시보드
/instructors             → 강사 리스트
/instructors/new         → 강사 생성
/instructors/:id/edit    → 강사 수정
```

---

## 8. 목 데이터

퍼블리싱 전용이므로 하드코딩된 목 데이터를 사용한다.

- 강사 50명분 목 데이터 생성
- 30개 분야에 랜덤 분배
- 성별 남:여 = 6:4 비율
- 등급 A:B:C:D = 10:15:15:10 비율
